# 1. Primeros pasos

## 🛠️ Software y Herramientas Utilizadas

- **PyCharm:** Entorno de Desarrollo Integrado (IDE) recomendado para escribir, depurar y estructurar código Python.
- **Jupyter Notebook:** Entorno interactivo basado en celdas, ideal para exploración de datos, visualización de tablas y pruebas rápidas de pipelines.
- **Git y GitHub:** Sistema de control de versiones local (Git) y plataforma en la nube (GitHub) para respaldar y colaborar en el repositorio.
- **UV:** Gestor de paquetes y entornos virtuales para Python escrito en Rust. Reemplaza herramientas tradicionales como `pip`, `virtualenv` y `poetry` ofreciendo una velocidad considerablemente superior.

> 💡 **Instalación en Arch Linux:**
> ```bash
> sudo pacman -S uv
> ```

## ⚡ Opción A: Creación manual del entorno virtual con UV

Esta opción es ideal si prefieres gestionar dependencias de forma clásica mediante un archivo `requirements.txt`.

### 1. Crear y ubicarse en el directorio del proyecto
```bash
cd ~/Documents
mkdir building_ETL_pipelines_with_python_book
cd building_ETL_pipelines_with_python_book
```

### 2. Generar y activar el entorno virtual
```bash
# Crear el entorno con la versión predeterminada del sistema
uv venv

# (Opcional) Crear el entorno especificando una versión concreta de Python
uv venv --python 3.12

# Activar el entorno virtual en la terminal (Linux/macOS)
source .venv/bin/activate
```

### 3. Gestión de dependencias
```bash
# Sincronizar dependencias si existe pyproject.toml / uv.lock
uv sync

# O instalar dependencias desde un archivo requirements.txt clásico
uv pip install -r requirements.txt

# Desactivar el entorno al finalizar el trabajo
deactivate
```

## 🚀 Opción B: Inicialización de un proyecto estructurado desde cero con UV

Esta opción es la recomendada para proyectos modernos, ya que genera automáticamente la estructura inicial del proyecto (`pyproject.toml`, `.gitignore`, `.git`, `README.md` y `main.py`).

### 1. Inicializar el proyecto con UV
```bash
cd ~/Documents
uv init building_ETL_pipelines_with_python_book
cd building_ETL_pipelines_with_python_book
```

### 2. Agregar dependencias al proyecto (opcional)
```bash
# Agrega paquetes y actualiza automáticamente el pyproject.toml y uv.lock, esto creará el entorno .venv con el python incluido, también se crea con uv run main.py
uv add pandas requests
```

### 3. Vincular y subir al repositorio de GitHub
```bash
# Conectar el repositorio local con GitHub
git remote add origin https://github.com/alvarorobleses/building_ETL_pipelines_with_python_book.git

# Crear la rama principal y realizar el primer envío
git branch -M main
git add .
git commit -m "feat: inicialización del proyecto ETL con uv"
git push -u origin main
```
